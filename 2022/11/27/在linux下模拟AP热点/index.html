在linux下模拟AP热点

Posted on 2022-11-27

相关工具

在Linux PC下使用WiFi网卡仿真一个AP热点需要如下3步，对应需要的组件分别为：

  • 创建WiFi AP：hostapd
  • 使能DHCP功能：udhcpd(或dnsmasq)
  • 使能NAT(路由转发)功能：iptables

如果热点下的设备不需要上网，则配置前两步即可。

操作步骤

前置条件

1.如果当前wifi网卡正在作为STATION模式使用，可能会导致多个进程同时操作wifi，出现冲突，建议先关闭network-manager（ubuntu默认安装的网络管理服务）：
sudo service network-manager stop
恢复方法：sudo service network-manager start
注意，关闭network-manager服务也会同时关闭有线网卡的使用，如果需要有线上网功能，也可以仅手动关闭wifi网卡。
2.如果之前开启的进程未完全结束，可能需要清除历史进程及缓存：

创建WiFi AP

  1. 新建配置文件test_hostapd.conf，可参考： 其中wlp4s0为wifi网卡在linux系统中的名称，可以通过ip addr或者ifconfig指令查看。
  2. 运行hostapd进程 说明：创建AP这一步，其实仅是让WiFi网卡拥有发送AP beacon和收发auth，assoc等管理帧的能力。
    此时如果有WiFi STA连上这台AP，还无法通过dhcp获取动态ip地址。

为创建的AP使能dhcp功能

上面已经提到，使能dhcp可以通过udhcpd或dnsmasq工具来实现，本文提供udhcpd的用法。

  1. 新建配置文件test_udhcpd.conf，可参考：
  2. 为AP设置ip地址，sudo ip addr add dev wlp4s0 192.168.30.1。
  3. 如果出现STA连上AP连上ARP不回复的情况，可以尝试修改路由表：sudo ip route add dev wlp4s0 default
  4. 运行udhcpd进程，sudo udhcpd -f test_udhcpd.conf &
