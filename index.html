ARM-主要寄存器

Posted on 2022-09-14

前言

寄存器是CPU架构的重要组成部分，要了解ARM架构，就必须了解主要的寄存器。

CPU中为什么要有寄存器

程序的执行可以分解成CPU的一条条基本操作，这些基本操作就是各种CPU指令：

  • 一条CPU指令通常借助多个缓存才能完成，比如一个相加的指令，就至少需要2个被加数，1个相加结果。
  • CPU需要一些缓存来保存自身的状态。

从以上描述可知，这些“CPU指令需要借助的缓存”应具有“可读取”和“可修改”的特性，这意味着缓存可以是内存，也可以是其他存储介质。但是内存访问每次都要通过数据总线去读取，显然会导致效率低下。
这种情况下寄存器应运而生，它们就是CPU可以随时快速使用的临时缓存空间。

ARM主要寄存器

ARM处理器共有7种不同的处理器模式，在每一种处理器模式中有一组相应的寄存器组。任意时刻(也就是任意的处理器模式下)，可见的寄存器包括15个通用寄存器(R0～R14)、一个或两个状态寄存器及程序计数器(PC)。在所有的寄存器中，有些是各模式共用的同一个物理寄存器，有一些寄存器是各模式自己拥有的独立的物理寄存器。

未分组寄存器(R0 - R7)

对于未分组寄存器，它们没有被系统用于特别的用途，因此任何可采用通用寄存器的应用场合都可以使用未分组寄存器。
但需要注意一点，未分组寄存器不会因为处理器模式的改变而更改指向的寄存器，因此在所有的处理器模式下未分组寄存器都指向同一个寄存器，当中断或异常处理造成处理器模式转换的时候，由于不同的处理器模式使用了相同的物理寄存器，这就有可能造成寄存器中的数据被破坏。

分组寄存器(R8 - R14)

对于分组寄存器，它们每一次所访问的物理寄存器和处理器当前的运行模式有关。例如在快速中断模式 fiq下R8-R12访问寄存器R8_fiq-R12_fiq；而在其他模式下又访问R8_usr-R12_usr。因此它们每个对应着两个不同的寄存器。

对于R13（SP）、R14（LR）来说，每个寄存器对应着6个不同的物理寄存器，其中的一个是用户模式与系统模式共用，另外5个物理寄存器对应于其他5种不同的运行模式。采用以下的记号来区分不同的物理寄存器：
R13_< mode >
R14_< mode >
其中，mode为以下几种模式之一：usr、fiq、irq、svc、abt、und。

R13(SP)

栈指针。存储栈地址，比如有个CPU，然后还有个外部资源也就是内存，我们想象CPU在程序跳转的时候在运行是最核心的ALU单元，然后外部资源存储着程序A和程序B还有程序C，实际上在CPU在读程序A的时候，可能下面的时候会跳到程序B单元，这个时候它可能不知道地址在哪里，那么这个时候就存储在SP这个寄存器里面，SP这个寄存器表示后面将要执行的程序。

R14(LR)

链接寄存器。存储于程序返回地址。这个链接寄存器主要用在函数A和函数B，A正在运行时，突然要调用B，那么就引了一个分支了，然后这个函数B去运行，运行完之后还是要返回到最初然后继续往下走，那么这个时候返回值应该要有个记录，这就是链接寄存器。

程序状态寄存器

R15(PC)

介绍R15之前先简单了解一下ARM处理器的是流水线机制。ARM7处理器采用3级流水线来增加处理器指令流的速度，能提供0.9MIPS/MHz的指令处理速度。

ARM7 的流水线有3个阶段，因此指令分3个阶段执行。
⑴ 取指从存储器装载一条指令
⑵ 译码识别将要被执行的指令
⑶ 执行处理指令并将结果写会寄存器

对于x86处理器来说，只有完成一条指令的读取和执行后，才会执行下一条指令。这样， PC 始终指向的正在“执行”的指令。
而对于 ARM7 来说因为是3级流水线，所以把指令的处理分为了上面所述的3个阶段。所以处理时实际是这样的： ARM 正在执行第1条指令的同时对第2条指令进行译码，并将第3条指令从存储器中取出。因此 ARM7 流水线只有在取第4条指令时，第1条指令才算完成执行。继而 ARM 的 PC 寄存器永远指向当前执行的指令后的第二条指令，即处于取指阶段的指令。

另外，在ARM状态下，最低的两位[1:0]为0，其他位[31:2]用于保存PC；在Thumb状态下，最低位[0]为0，其他位 [31:1]用于保存PC；所以 R15（PC）虽然可以用作通用寄存器，但是有一些指令在使用R15时有一些特殊限制，当违反了这些限制时，程序的执行结果是未知的。

R16(CPSR/APSR)

寄存器R16只在usr下面被叫做APSR（应用程序状态寄存器），其他情况下叫做CPSR（Current Program Status Register）。
R16可在任何运行模式下被访问，它包括条件标志位、中断禁止位、当前处理器模式标志位，以及其他一些相关的控制和状态位。

R17(SPSR)

每一种运行模式下又都有一个专用的物理状态寄存器，称为备份的程序状态寄存器 SPSR （Saved Program Status Register），当异常发生时， SPSR 用于保存 CPSR 的当前值，从异常退出时则可由 SPSR 来恢复 CPSR 。

由于用户模式和系统模式不属于异常模式，他们没有 SPSR ，当在这两种模式下访问 SPSR ，结果是未知的。

操作系统-原子操作与互斥操作

Posted on 2022-09-08

背景

如果多个线程访问相同的数据，一个正在读取共享数据的线程可能中断另一个正在修改相同共享数据的线程，导致线程读取到不一致的数据。
为了避免多线程同时访问同一块数据，可以用原子操作或者互斥操作进行相应的保护。

互斥操作

互相排斥（mutex exclusion）技术，简称为互斥（mutex），它用于防止多个线程同时访问共享资源。互斥技术采用一个对象控制独占访问权限，该对象称之为互斥。配合条件变量（condition variable），互斥可以实现广泛的同步访问控制。例如，它们允许程序员为数据访问操作指定执行次序。

原子操作

原子操作，指CPU操作过程中不能被中断。原子曾被认为是不可分割的最小单元，原子操作也可以被理解为一个不可被分割的操作。在针对某个值进行原子操作时，CPU绝对不会再进行其他的针对该值的操作，通常，原子操作仅会由单条CPU指令完成。原子操作是无锁的，其他同步技术实现也常依赖于原子操作。

互斥操作和原子操作的区别

  • 互斥操作时，线程取锁之后，可以被其他线程打断再返回回来执行；而原子操作是单个不可被打断的操作，通常是由单条CPU指令完成，所以没有其他线程可以打断它。
  • 原子操作的优势：更加轻量，可以在不形成临界区的情况下完成并发安全的值得替换操作。可以大大减少同步对程序性能的消耗。
  • 原子操作的劣势：原子操作是乐观锁，总是假设操作未被改变，在被操作的值被频繁改变的情况下，原子操作并不那么容易成功；而互斥锁是悲观锁，我们总是假设会有并发的操作要修改被操作的值，因此使用锁对一系列操作加上临界区保护。

ARM-CPU核间通信如何实现

Posted on 2022-09-07 Edited on 2022-09-08

问题

对于多核的CPU架构，如果两个核之间需要进行通信，如何实现？

实现原理

核间通信，又叫Inter Processor Communication(IPC)。
这里仅简单介绍一种方式，基本的思想是“共享内存+互斥操作+通知机制”：

  • 共享内存，即两个核能访问同一块内存区域。
  • 互斥操作，即两个核不会在同一时刻去修改同一个内存区域。
  • 通知机制，即一个核通过GPIO中断或其他方式来通知另一个核有消息要发送。

当A核需要向B核发送消息时，大致过程为：

  • 首先A核通过互斥操作取得共享内存的修改权；
  • 然后A核将要发送的消息放置到共享内存中；
  • 最后A核通过通知机制告诉B核来读取内存中的数据。

ARM-芯片架构简介

Posted on 2022-09-05 Edited on 2022-09-07

简介

ARM(Advanced RISC Machine)架构。是一个精简指令集(RISC,Reduced Instruction Set Computer)处理器架构家族，具有低成本、高性能、低耗电的特性。
ARM架构广泛应用在嵌入式系统，现在也有应用在个人计算机（Apple M1）和超级计算机中。

  • 与ARM同属精简指令集架构的还有RISC-V，MIPS，Xtensa等。
  • 与精简指令集相对应的是复杂指令集（CISC,Complex Instruction Set Computer），代表架构为X86。

ARM架构范畴

ARM架构的主要范畴整理如下：

条目 说明
指令集 每条指令的功能；指令如何在内存中表示(它的编码)
寄存器组 有多少寄存器；寄存器的大小；寄存器的功能；寄存器的初始状态
异常模型 不同的权限级别；异常的类型；从异常中获取或返回时会发生什么
内存模型 内存访问是如何排序的；缓存是如何运作的，什么时候以及软件必须如何执行显式维护
调试、跟踪和分析 如何设置和触发断点；跟踪工具可以捕获什么信息，以什么格式捕获
安全性扩展(TrustZone) 安全性扩展，隔离某些资源

ARM-芯片启动流程

Posted on 2022-09-05

芯片启动流程

在MCU(CPU)上执行C语言编译的可执行文件时，基本可以将启动流程分为以下3个阶段：

  • 初始化阶段
  • 执行阶段
  • 退出阶段

ARM芯片也不例外，仅与其他架构芯片有一些细节上的不同。

初始化流程

硬件初始化

硬件初始化通常在启动代码cstartup.s中执行，如果需要，还可以通过用户提供的额外低级接口__low_level_init来执行用户自己的代码。它可能包括重置/启动其余硬件，设置CPU等，以准备软件c/c++系统初始化。

  • 当cpu复位的时候，，程序将从系统启动代码中的程序入口entry开始执行。
  • 堆栈指针初始化为 CSTACK 块的末尾。
  • 对于Arm7/9/11、Cortex-A和Cortex-R器件，异常堆栈指针初始化为每个相应部分的末尾。
  • 如果用户定义了函数__low_level_init，则为应用程序提供执行早期初始化的机会。

c/c++软件系统初始化

通常，此步操作确保在调用 main 函数之前，每个全局（静态链接）的 C/C++符号都会收到其正确的初始化值。

  • 静态和全局变量初始化。也就是说，清零初始化为零的变量，并将其他初始化变量的值从 ROM 复制到 RAM 存储器。 如果__low_level_init返回零，则跳过此步骤。
  • 在标准 C 中，所有静态变量（在固定内存地址分配的变量）必须由运行时系统初始化为应用程序启动时的已知值。此值是分配给变量的显式值，或者如果没有给出值，则将其清零。在编译器中，此规则有例外，例如声明为__no_init的变量，它们根本没有初始化。
    • 对于已经初始化的数据，从ROM复制到RAM
    • 对于未初始化的数据，用零填充
  • 构造静态 C++ 对象。
  • 调用主函数，启动应用程序。

应用程序初始化

就是用户自己的程序部分。这完全取决于用户的应用程序：对于带操作系统的应用程序，它可以包括设置RTOS内核并启动RTOS驱动的应用程序的初始任务；对于裸机应用程序，它可以包括设置各种中断，初始化通信，初始化设备等。
对于基于 ROM/flash 的系统，常量和函数已经放在 ROM 中。 放置在 RAM 中的所有符号必须在调用main函数之前初始化。链接器已将可用RAM划分为变量，堆栈，堆等的不同区域。

  • 当应用程序启动时，系统启动代码首先执行硬件初始化，例如初始化栈顶指针以指向预定义堆栈区域的末尾：

执行阶段

这里的执行阶段指的是应用程序的执行。
应用程序可以是无操作系统，也可以是有操作系统。

退出阶段

大多数情况，应用程序是在一个循环中执行，不涉及退出。
如果一定要中途退出main函数，有两种方法：

  • 在main函数中执行return；
  • 执行exit()；

FreeRTOS源码精读-tasks

Posted on 2022-09-05 Edited on 2022-09-07

阅读原则

tasks.c是FreeRTOS的核心代码，虽不算非常复杂，也有5000多行代码，一行行代码看的话一天也不够用。我推荐的阅读步骤是：

  • 先看API、数据结构、全局变量
  • 再看主要函数的源码
  • 最后看次要函数的源码

关键API

xTaskCreate

功能：申请动态内存，并创建任务，为任务结构体（TCB_t）赋初值。
这个函数仅在支持动态内存申请的宏使能时生效。

值得注意的是，有的CPU的栈内存是从低向高增长，有的CPU栈内存是高向低增长（如ARM架构）。这会影响任务控制块内存（pxNewTCB）和任务内存（pxStack）的申请顺序，对于一块全新的内存来说，malloc通常是优先申请低地址的内存，再申请高地址的内存。

两种不同栈生长方向的内存申请情况如下图所示：

为了避免栈增长到溢出的时候踩踏到pxNewTCB的内存，freeRTOS实现了一套任务控制块内存保护机制：

  • 若CPU的内存是由高向低增长，则先申请任务内存（pxStack），再申请任务控制块内存（pxNewTCB），这样pxNewTCB就处于pxStack的更高地址，栈顶指针pxTopOfStack的初始值就放在任务内存（pxStack）的最高地址（为了向低增长），也就是内存数组的结尾处。
  • 此后，任务内存（pxStack）不断被使用，pxTopOfStack也逐渐指向更低的地址，但再也不会和任务控制块内存相遇。这就降低了自身控制块内存被自身stack内存踩踏的风险。
  • 值得注意的是，由于内存是由malloc去分配的，当内存被多次循环使用并释放后，并不能保证后申请的内存地址就比先申请的内存地址更高（也就是说，前文说提的任务控制块内存保护机制并不总能生效），也不能保证连续申请的两块内存物理地址是连续的（此时这种内存保护机制无需生效）。

xTaskCreateStatic

功能：以静态内存创建任务
这个函数与xTaskCreate的区别仅在于任务相关的内存的申请方式不通，该函数用的是预先指定的静态内存。

prvInitialiseNewTask

功能：初始化一个新的任务，被以上两个APIxTaskCreate和xTaskCreateStatic调用。

  • 重置任务内存（pxStack）为默认值，默认为0xA5
  • 根据栈增长方向设置栈顶指针pxTopOfStack的初始值
  • 初始化优先级、任务名。
  • 初始化xStateListItem，xStateListItem是一个重要的元素，其类型为一个链表的节点，挂在不同的链表（就绪，阻塞，删除，挂起）下时，意味着其处于不同的任务状态。
  • 初始化xStateListItem

vTaskDelete

功能：删除任务

prvInitialiseTaskLists

功能：初始化各个状态的任务链表全局变量，在创建第一个任务的时候会被调用。

vTaskStartScheduler

功能：开启任务调度器。

  • 创建第一个任务（或称线程），idle任务
  • （如果使能了timer宏）创建定时器任务
  • 开始调度器（xPortStartScheduler），初始化tick时钟，tick是整个操作系统的心跳。

vTaskEndScheduler

功能：结束任务调度器。（很少使用）

其他

  • 任务按相对时间延迟vTaskDelay、任务按绝对时间延迟xTaskDelayUntil；
  • 任务优先级设置vTaskPrioritySet、任务优先级获取uxTaskPriorityGet，uxTaskPriorityGetFromISR;
  • 挂起任务vTaskSuspend、恢复任务vTaskResume；
  • 挂起所有任务vTaskSuspendAll，恢复所有任务xTaskResumeAll，支持递归调用。
  • 任务上下文切换vTaskSwitchContext
  • 任务优先级继承xTaskPriorityInherit（与互斥锁有关）
  • 当前所有任务信息获取vTaskList

  • xTasksWaitingTermination，被删除（但还未删除内存的）任务列表；
  • xSuspendedTaskList，挂起任务列表；

理解了这几个全局变量，就能轻松地理解task.c中的函数对于任务状态的切换过程。

小结

理解了最关键的几个全局变量、数据结构、任务函数，就能快速掌握task.c的核心功能。

网络协议-不同MTU的网卡如何进行TCP通信

Posted on 2022-09-04

问题

一台路由器下连接了两个子设备，一台设备的网卡MTU（最大传输单元）是1500，一台设备的MTU是1000，如何进行TCP通信，在哪一层协商？

分析

要解决这个问题，最基本的两个知识是TCP分段和IP分片，简单来说：

  • 分段特指发生在使用TCP协议的传输层中的数据切分行为
  • 分片特指发生在使用IPv4协议的网络IP层中的数据切分行为

TCP分段

TCP分段的原因是因为TCP报文段大小受MSS限制。

最大报文段长度MSS（Maximum Segment Size），为TCP传输层的最大载荷上限(即应用层数据最大长度)。
MSS是TCP的报头的可选项，MSS的协商在TCP握手阶段完成：

B将自己的MSS发送给A，建议A在发数据给B的时候，采用MSS=1420进行分段。而B在发数据给A的时候，同样会带上MSS=1372。两者在对比后，会采用小的那个值（1372）作为通信的MSS值，这个过程叫MSS协商。

通常一条TCP连接的MSS取通信双方较小的那一个MSS值，与MTU的换算关系为：
MTU = MSS + TCP首部长度 + IP首部长度
故在以太网中(网络层以IPv4为例)：
MSS = 以太网MTU - TCP首部长度 - IPv4首部长度 = 1500 - 20 - 20 = 1460字节

为什么MSS的大小需要协商呢？还得从IP层分片说起。

IP层分片

IP分片的原因是因为IP数据报大小受MTU限制。

最大传输单元MTU(Maximum Transmit Unit)，这个值是由数据链路层提供，为了告诉上层IP层，自己的传输能力是多大。
受各种因素限制，一个网卡能发出的单包数据长度是有限的，这个单包数据的最大长度就是数据链路层的传输能力。

发送端的IP分片

在网络数据发送者开始发送时，IP层就会根据MTU进行数据包切分，一般MTU=1500 Byte。

  • 假设IP层有 <= 1500 Byte 需要发送，只需要一个IP包就可以完成发送任务；
  • 假设IP层有 > 1500 Byte 数据需要发送，需要分片才能完成发送，分片后的 IP Header ID 相同，同时为了分片后能在接收端把切片组装起来，还需要在分片后的IP包里加上各种信息。比如这个分片在原来的IP包里的偏移offset。

传输路径上的IP分片

我们知道在因特网通信过程，网络数据在很多路由器节点之间跳转，而各个节点间MTU可能不同，这就导致数据是可能被多次分片的。而且每次分片都要加上各种信息便于在接收端进行分片重组。
如果有办法知道整个链路上，最小的MTU是多少，并且以最小MTU长度发送数据，那么不管数据传到哪个节点，都不会发生分片。
整个链路上，最小的MTU，就叫PMTU（path MTU）。
有一个获得这个PMTU的方法，叫 Path MTU Discovery。
Path MTU Discovery的大致原理是：

通过向对端发送申明不要分包(DF)的IP报文，然后根据是否收到传输路径上节点的“ICMP不可达报文”，来判断目前的发送数据长度是否过长，如果过长则减少长度重新试探，直至发送成功。

小结

不同MTU的两个网卡设备进行TCP通信时，两端先根据自身的MTU计算出MSS，然后在TCP握手的时候进行MSS协商。

读书笔记-《红拂夜奔》

Posted on 2022-09-03 Edited on 2022-09-04

作者：王小波
阅读时间：2022.03.22

这是一本以明代戏剧《红拂传》人物为原型创作的小说：
李靖、红拂、虬髯公，世称风尘三侠。隋朝末年，他们三人都在洛阳城里住过。

  • 李靖解开费马定理，证明了自己是一个聪明人。他成为唐朝开国功臣“李卫公”后，却装傻到老。
  • 红拂曾是歌姬，与李靖私奔后成为了他的妻子——唐朝“第一夫人”，丈夫死后她无奈上吊自杀。
  • 虬髯公深爱着红拂，求而不得，后成为了扶桑国的国王，过着扭曲的生活。

这个故事荒诞得好像现实。我们很容易从书中的某个事物看到现实中的影子。
《红拂夜奔》的内容太多，讽刺的密度之高令人叹为观止，我无法也不便一一列出。可以确认的是，王小波对于这个世界看得非常地清楚。

这本书也许能改变一些人的世界观：你以为书里的世界很离谱，殊不知现实远比书中荒唐一百倍。

You have to know the past to understand the present.

读书笔记-《罗素谈人的理性》

Posted on 2022-09-03

作者：罗素
阅读时间：2020.12.05

读罗素的书是因为王小波对他的推崇，而我个人也喜欢看一些哲学相关的书籍。
这也是一本影响了我个人世界观的书籍，推荐。
当时没有写完整的书评，仅在微信读书上记录了一些划线笔记，现搬移过来，暂不做整理。

第1章 人有不幸

我认为，这种不幸在很大程度上是由对世界的错误看法、错误的伦理观、错误的生活习惯所引起的，结果导致了对那些可能获得事物的天然热情和追求欲望的丧失，而这些事物，正是人和动物的一切幸福、快乐最终有赖于它们的。

王小波说人的一切痛苦来自于对无能的愤怒，而罗素眼中，大概还有错误的世界观。在我看来，错误也是无知的一种，王小波是了解罗素的，不知他是如何看待“无能”和“无知”之间的关系的。

第5章 疲劳的工作

但是，我们有必要认识到，使人兴奋的娱乐并非通向幸福之路，尽管只要使人更满足的快乐是可望而不可及的，一个人会发现，除非通过兴奋的刺激，否则生活是难以忍受的。

过度娱乐带来疲劳，长久的平淡反而使人清醒。

第9章 舆论与恐惧

在现代世界，由于交通工具的创造发明，人们不必再像过去那样，仅仅与自己的近邻来往了。那些拥有汽车的人可以把方圆20英里内的任何人看做自己的邻居。因此他们比过去有更多的机会来选择自己的朋友。

有趣的现象，在中国和美国的城市化进程中，都出现了邻里变得陌生的情况。但个人认为其主要原因并不是交通工具的出现，而是由人在社会中的角色变化。其一，现代社会功能的强大使得人不必再依靠邻里的帮扶和受到宗族村落的制约。其二，现代的工作时间特性使得人们早出晚归，邻里之间鲜有机会和精力见面，人们更愿意按自己的喜好选择多种多样的消遣方式。

第11章 渴望的热情

一个人的兴趣越广，他拥有的快乐机会就越多，而受命运摆布的可能性也就越小，因为如果他失去了某一种兴趣，他便可转而依赖另一种。生命短暂，人们不可能事事都感兴趣，不过对尽可能多的事物感兴趣却是一桩好事。我们都容易染上内省者的弊病，世界向他吐现出千姿百态的景象，但他却别转脑袋，专注于内心的空虚，我们可不要以为，内省者的忧郁有什么了不起之处。

第14章 时代与劳务

一项伟大的建设性事业所给予的乐趣是人生所能奉献的最大的快慰之一，虽然不幸得很，这种卓绝超群的欢乐仅为不同凡俗的人所享受。

哲学的魅力在于，将一些事情、道理抽象出最本质的部分，而这又和写代码的过程很相似。解构并重塑，能让一件事情达到新的高度。

第20章 休闲生活

富人和奉承他们的人发表赞美可敬的劳苦工作，颂扬简朴的生活，宣传一种教义，那就是所谓穷人比富人容易进入天堂。总之，他们竭力使体力劳动者相信改变物质在空间中的形式及其位置的工作是特别高尚的，所有这些就同男人竭力让女人相信她们之所以特别高尚在于她们性的约束一样。

所有事物背后必有其道理。儒家思想控制士大夫阶层忠君爱国，得以被统治阶级发扬光大。类似的情况在古今中外都是如此的一致。
很多事情你看到是这样子，不是因为它本身是这样子，而是因为有人希望你看到是这样子。
这个世界是怎么样的，一家之言永远是肤浅的。读万卷书，行万里路才是最好的答案。终身学习的可贵之处也在这里。
